# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Credo, please report it privately so it can be fixed before public disclosure.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, use GitHub's private vulnerability reporting:

1. Open the **[Security](https://github.com/credo-go/credo/security)** tab of the repository.
2. Click **"Report a vulnerability"** to start a private advisory.

Please include:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- A suggested fix, if you have one

## What to expect

Credo is maintained on a best-effort basis. Security reports are reviewed, acknowledged as soon as is practical, and triaged by severity — critical issues are prioritized above other work. There is no fixed response-time guarantee, but a genuine report will not be ignored, and you will be kept informed of progress toward a fix.

## Supported Versions

Until the first stable release (`v1.0.0`), only the latest tagged release receives security fixes.

| Version             | Supported |
| ------------------- | --------- |
| Latest release      | Yes       |
| Older / pre-release | No        |

## Disclosure Policy

Credo follows coordinated disclosure. Please give the maintainers a reasonable opportunity to release a fix before disclosing publicly. Reporters are credited when the fix is published, unless they request anonymity.
